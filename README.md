# KSU Academic Network

Streamlit-сервіс для обліку, перегляду та аналітики наукових публікацій викладачів KSU на базі Neo4j Aura.

## Що вже реалізовано

- дашборд із ключовими метриками, структурою та покриттям профілів;
- каталог викладачів і публікацій;
- граф авторства, співавторства та міжкафедральних зв'язків;
- аналітика, рейтинги, centrality і звіти;
- центр даних для модерації, аудиту та ручного додавання;
- імпорт публікацій через `ORCID`, `OpenAlex`, `Crossref`, `Scopus`, `Web of Science`, `Google Scholar`;
- публічний і адміністративний режими.

## Режими роботи

### Публічний режим

Видимі сторінки:

- `Дашборд`
- `Граф`
- `Аналітика`
- `Викладачі`
- `Публікації`

У цьому режимі прихована адмінка, а таблиці не показують внутрішні ID.

### Адміністративний режим

Додатково відкриваються:

- `Структура`
- `Центр даних`

Саме тут доступні:

- імпорт і оновлення структури;
- ручне додавання та редагування;
- модерація публікацій;
- зміна статусів;
- аудит;
- видалення та масові дії.

Рекомендований спосіб доступу до адмінки: пароль у `Streamlit Secrets`.

```toml
ADMIN_PASSWORD = "your-password"
```

Після цього в sidebar з'являється блок `Режим керування`, який розблоковує адмінсторінки тільки для поточної сесії браузера.

`ADMIN_MODE=true` варто використовувати лише як аварійний варіант, якщо потрібно примусово відкрити адмінрежим для всіх відвідувачів.

## Запуск

```bash
streamlit run app.py
```

## Secrets для Streamlit Cloud

```toml
NEO4J_URI = "neo4j+s://..."
NEO4J_USER = "neo4j"
NEO4J_PASSWORD = "..."

OPENALEX_API_KEY = "..."
CROSSREF_MAILTO = "you@example.com"
ORCID_CLIENT_ID = "..."
ORCID_CLIENT_SECRET = "..."

# optional
SCOPUS_API_KEY = "..."
SCOPUS_INSTTOKEN = "..."
WOS_API_KEY = "..."
PUB_IMPORT_MAX_WORKS_PER_TEACHER = 40

# public/admin mode
ADMIN_PASSWORD = "your-password"
# optional emergency switch
ADMIN_MODE = false
```

`NEO4J_DATABASE` необов'язковий. Для `Neo4j Aura` його краще не задавати, якщо домашня база вже визначається автоматично.

## Структура проєкту

- `app.py` — точка входу, маршрутизація сторінок і перемикання public/admin mode.
- `config.py` — secrets, env і конфігурація зовнішніх джерел.
- `views/` — сторінки Streamlit:
  - `dashboard.py`
  - `teachers.py`
  - `publications.py`
  - `graph.py`
  - `analytics.py`
  - `structure.py`
  - `data_center.py`
- `ui/` — тема, компоненти, sidebar і форматтери таблиць.
- `services/neo4j_service.py` — основна бізнес-логіка, CRUD, графові запити, модерація, аудит.
- `services/publication_import.py` — оркестрація імпорту публікацій.
- `services/publication_sources.py` — робота з джерелами та матчингом.
- `utils/` — аналітика та графова візуалізація.
- `data/` — seed-дані та шаблони імпорту.
- `docs/data_pipeline.md` — опис логіки імпорту й верифікації.
- `scripts/` — допоміжні скрипти підготовки даних.

## Основні сторінки

- `Дашборд` — загальний стан системи, структура, покриття, джерела.
- `Викладачі` — пошук по викладачах, картка, публікації, співавтори.
- `Публікації` — публічний каталог публікацій з фільтрами.
- `Граф` — мережеві візуалізації на базі Neo4j.
- `Аналітика` — топи, centrality, звіти кафедр і факультетів.
- `Структура` — адміністративне керування факультетами, кафедрами та викладачами.
- `Центр даних` — модерація, аудит, підозра на дублікати, ручне додавання.
